作用域与作用域链

1. 全局作用域

// 全局作用域 const globalVar = "全局变量"; console.log("全局作用域中的 globalVar:", globalVar);
执行结果：

全局作用域的特点：

  • 在代码的任何地方都可以访问
  • 使用 var 声明的变量会成为全局对象的属性
  • 在浏览器环境中，全局对象是 window
  • 全局作用域的变量会在页面关闭时才被销毁

2. 函数作用域

function outer() { // 函数作用域 const outerVar = "外部函数变量"; function inner() { // 函数作用域 const innerVar = "内部函数变量"; console.log("内部函数访问:", globalVar, outerVar, innerVar); } inner(); // console.log(innerVar); // 会报错，innerVar 只在 inner 函数内有效 } outer(); // console.log(outerVar); // 会报错，outerVar 只在 outer 函数内有效
执行结果：

函数作用域的特点：

  • 在函数内部声明的变量，只在函数内部有效
  • 函数内部可以访问外部作用域的变量
  • 函数外部不能访问函数内部的变量
  • 函数作用域会在函数执行完毕后被销毁

3. 块级作用域

// 块级作用域 if (true) { const blockVar = "块级变量"; console.log("块内访问 blockVar:", blockVar); } // console.log(blockVar); // 会报错，blockVar 只在块内有效
执行结果：

块级作用域的特点：

  • 使用 let 和 const 声明的变量，在块级作用域内有效
  • 块级作用域包括 if、for、while、switch 等语句块
  • 块级作用域会在代码块执行完毕后被销毁
  • var 声明的变量没有块级作用域，只有函数作用域

4. 作用域链

作用域链的概念：

  • 当在一个作用域中访问变量时，会先在当前作用域中查找
  • 如果找不到，会向上查找父级作用域
  • 一直查找到全局作用域
  • 如果全局作用域中也找不到，会返回 undefined
  • 这种由内向外查找变量的链条，称为作用域链

作用域链的示例：

  • 内部函数可以访问外部函数的变量
  • 外部函数不能访问内部函数的变量
  • 多个嵌套函数形成一条作用域链
