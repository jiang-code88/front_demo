变量提升与暂时性死区

1. var 变量提升

// var 变量提升 console.log("var 变量提升:", varVar); // undefined (变量提升，但未赋值) var varVar = "var 变量"; console.log("var 变量赋值后:", varVar); // var 变量
执行结果：

var 变量提升的特点：

  • 变量声明会被提升到作用域的顶部
  • 变量赋值不会被提升
  • 在赋值前访问变量会返回 undefined

2. let 和 const 的暂时性死区

// let 和 const 不会变量提升，存在暂时性死区 // console.log("let 变量提升:", letVar); // 会报错，暂时性死区 let letVar = "let 变量"; console.log("let 变量赋值后:", letVar); // let 变量
执行结果：

暂时性死区的特点：

  • let 和 const 声明的变量不会被提升
  • 在变量声明之前的区域称为暂时性死区
  • 在暂时性死区内访问变量会抛出错误

3. 函数声明的提升

// 函数声明会变量提升 console.log("函数声明提升:", hoistedFunc()); // 函数被提升 function hoistedFunc() { return "函数被提升"; }

4. 函数表达式不会提升

// 函数表达式不会变量提升 // console.log("函数表达式提升:", nonHoistedFunc()); // 会报错，nonHoistedFunc 是 undefined const nonHoistedFunc = function() { return "函数表达式"; }; console.log("函数表达式赋值后:", nonHoistedFunc()); // 函数表达式
执行结果：

函数表达式的特点：

  • 函数表达式不会被提升
  • 在赋值前访问函数表达式会返回 undefined
  • 箭头函数也是函数表达式的一种，同样不会被提升

5. 变量提升与暂时性死区总结

变量提升规则：

  • var 声明的变量会被提升，赋值不会被提升
  • let 和 const 声明的变量不会被提升，存在暂时性死区
  • 函数声明会被提升，函数表达式不会被提升

最佳实践：

  • 使用 let 和 const 代替 var
  • 在作用域的顶部声明所有变量
  • 避免在变量声明前访问变量
