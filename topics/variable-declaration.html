变量声明：var、let、const 的区别与作用域

1. var 声明

// var 声明 var a = 10; var a = 20; // 可以重复声明 console.log("var 重复声明后的值:", a); // 输出 20 // 函数作用域 function testVar() { var b = 30; if (true) { var b = 40; // 同一函数作用域内，覆盖外部的 b console.log("函数内 if 块中的 b:", b); // 输出 40 } console.log("函数内的 b:", b); // 输出 40 } testVar();
执行结果：

var 的特点：

  • 可以重复声明同一个变量
  • 函数作用域，在函数内部任何位置声明的变量，在整个函数内都可访问
  • 存在变量提升现象，在 var 声明语句之前可以访问到变量，不会报错，只是值是 undefined（而不是报错）。

2. let 声明

// let 声明 let c = 50; // let c = 60; // 不能重复声明，会报错 // 块级作用域 if (true) { let d = 70; console.log("if 块中的 d:", d); // 输出 70 } // console.log("if 块外的 d:", d); // 会报错，d 只在块内有效
执行结果：

let 的特点：

  • 不能重复声明同一个变量
  • 块级作用域，只在声明的块或子块中有效
  • 不存在变量提升，存在暂时性死区，只能在声明后才能访问到变量，否则会报错。

3. const 声明

// const 声明 const e = 80; // e = 90; // 不能修改值，会报错 // const 声明对象 const obj = { name: "张三" }; obj.name = "李四"; // 可以修改对象属性 console.log("修改后的 obj:", obj); // 输出 { name: '李四' }
执行结果：

const 的特点：

  • 声明后不能修改值（基本类型）
  • 可以修改对象的属性值
  • 块级作用域
  • 不存在变量提升，存在暂时性死区，只能在声明后才能访问到变量，否则会报错。
  • 实际开发中尽量使用 let 和 const，它们没有变量提升带来的奇怪行为，也更符合块级作用域的直觉。
