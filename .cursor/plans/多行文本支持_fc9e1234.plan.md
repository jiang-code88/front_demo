# 条目值支持多行文本

## 涉及文件
- [`tampermonkey-autofill.user.js`](tampermonkey-autofill.user.js) — 唯一文件，全部改动集中于此

---

## 改动点清单

### 1. 候选面板 — 条目值展示样式（第 427-433 行）

**当前问题**：`.afh-item-val` 设置了 `white-space: nowrap`，导致多行内容被强制压缩为一行，`text-overflow: ellipsis` 进一步将超出部分截断为省略号，用户在面板中看不到完整的换行内容。

**改动**：使用 `-webkit-line-clamp` 实现"多行截断 + 省略号"效果，限制最多显示 **3 行**，超出部分显示 `…`：

```css
.afh-item-val {
  display: -webkit-box;
  -webkit-box-orient: vertical;
  -webkit-line-clamp: 3;
  overflow: hidden;
  white-space: pre-wrap;   /* 保留换行符 */
  word-break: break-all;
}
```

**展示效果变化**：
- 单行内容：与之前一致，仍展示为一行
- 2-3 行内容：完整多行展示，无截断
- 超过 3 行的大段文本：仅显示前 3 行，末尾追加 `…` 省略提示，条目高度保持可控，不会过于臃肿
- `title` 属性保留完整内容，鼠标悬停时可通过 tooltip 查看全文（已有代码 `val.title = item.value`）

---

### 2. 候选面板 — 内联表单 value 输入框样式

**当前问题**：`.afh-finput` 固定 `height: 27px`，如果将值输入框替换为 `<textarea>`，固定高度会使 textarea 只显示一行且无法拖拽调整，体验极差。

**改动**：新增 `.afh-finput-ta` 样式，在继承 `.afh-finput` 外观（边框、圆角、字号、颜色）的基础上，去掉固定高度，改为 `min-height: 54px; resize: vertical; padding: 4px 8px;`（允许用户拖拽调整高度）。

**展示效果变化**：值输入框从单行变为多行文本区域，默认显示约两行高度，用户可以自由换行输入，也可以拖拽下边缘调整高度。

---

### 3. 候选面板 — `buildForm()` 值输入改为 textarea（第 1100-1104 行）

**当前问题**：`valInput` 是 `<input type="text">`，单行输入框不支持换行，用户按下 Enter 键会直接触发保存，而不是换行。

**改动**：
- `valInput` 由 `createElement('input')` → `createElement('textarea')`，应用 `afh-finput afh-finput-ta` 样式
- **移除** `valInput.addEventListener('keydown', onEnter)`：`<input>` 中 Enter 触发保存是合理的，但 `<textarea>` 中 Enter 应该换行，不能触发保存；仅保留 `lblInput` 的 Enter 保存绑定

**展示效果变化**：在候选面板底部的快速添加/编辑区，"填入内容"一栏从单行输入框变为可换行的多行文本区。输入时按 Enter 换行，按 Tab 或点击"保存"按钮提交。

---

### 4. 设置页 — 弹窗表单 value 输入改为 textarea（第 1837-1840 行）

**当前问题**：设置页新增/编辑弹窗中，"填入内容"字段是 `<input class="s-input">`，同样是单行输入，不支持多行编辑。

**改动**（注意：此处是字符串拼接代码，即 `buildSettingsScript()` 内）：
- `valIn` 由 `createElement("input")` → `createElement("textarea")`
- class 由 `s-input` → `s-textarea s-input-ta`
- 新增 `.s-input-ta` 样式，覆盖 `.s-textarea` 中专为导入导出设计的 `height:90px` 和 `font-family:monospace`，改为 `min-height:60px; height:auto; resize:vertical; font-family:inherit`（字体与表单其他字段保持一致）
- **移除** `valIn.addEventListener("keydown", onEnter)`，原因同第 3 点

**展示效果变化**：设置页弹窗中"填入内容"字段变为多行文本区，与标签、范围字段在视觉上有明显区分，支持粘贴和输入多行内容，高度可拖拽调整。

---

### 5. 设置页 — 数据表格值列展示（第 1703-1705 行）

**当前问题**：表格"填入内容"列使用内联样式 `white-space:nowrap; overflow:hidden; text-overflow:ellipsis`，多行内容被压缩为一行并截断，用户无法直观看到值中是否包含换行内容。

**改动**：`tdV.style.cssText` 改为 `max-width:300px; overflow:hidden; white-space:pre-wrap; word-break:break-all; vertical-align:top;`，同时新增 `vertical-align:top` 让多行内容从顶部对齐，与同行其他列（标签、范围、操作按钮）对齐更美观。

**展示效果变化**：原来表格中每行"填入内容"列只显示一行文本末尾加省略号，改后会展开为多行文本，每行高度随内容行数自动增大，换行位置与存储内容完全一致。

---

## 特别说明

- **自动填充行为**：`fillInput()` 已正确处理 `textarea` 和 `input` 两种目标元素。对于 `<textarea>` 目标，`\n` 换行符会被原样填入；对于 `<input>` 目标，浏览器会自动忽略换行符，仅保留文字内容，这是浏览器标准行为，无需额外处理。
- **保存逻辑**：`valInput.value.trim()` 仅去除字符串首尾的空白（含换行），内部换行符完整保留，数据存储和读取均无需修改。
