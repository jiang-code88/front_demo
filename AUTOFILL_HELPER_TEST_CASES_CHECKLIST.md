# AutoFill Helper — 手工测试用例（实用 Checklist 版）

> 脚本版本：1.0.0  
> 最后更新：2026-06-05  
> 使用说明：每次发版前逐行过一遍，在"结果"列打 ✅（通过）或 ❌（失败），失败时在"备注"列记录原因  
> 操作步骤不清楚时查：[AUTOFILL_HELPER_TEST_CASES_STANDARD.md](./AUTOFILL_HELPER_TEST_CASES_STANDARD.md)

---

## 优先级说明

| 优先级 | 定义 | 适用场景 |
|--------|------|---------|
| **P0** | 核心链路，必测。任意一条失败即阻断发版 | 紧急修复发版、所有正式发版 |
| **P1** | 常用功能，应测。失败需评估影响后决定是否发版 | 正式发版、功能性改动后回归 |
| **P2** | 边界场景，可选。失败可记录为已知问题延后修复 | 有充裕时间时全量回归 |

> **快速回归（P0 only）**：只跑标记为 P0 的条目，约 15 条，预计 5–8 分钟。  
> **正式发版回归**：跑 P0 + P1，约 50 条，预计 15–20 分钟。  
> **全量回归**：跑全部 68 条，预计 25–30 分钟。

## 测试环境配置

### 必要条件
| 项目 | 要求 |
|------|------|
| 浏览器 | Chrome 90+ 或 Edge 90+（推荐 Chrome 最新稳定版） |
| Tampermonkey | v4.16 及以上 |
| 脚本状态 | 已安装并启用 `tampermonkey-autofill.user.js` |

### 测试前：准备基础数据
每次执行测试前，建议先通过设置页导入以下标准测试数据，确保所有条目相关用例有数据可用：

```json
{
  "global": [
    { "id": "test_g1", "label": "全局-姓名", "value": "张三", "createdAt": 1700000000000 },
    { "id": "test_g2", "label": "全局-邮箱", "value": "zhangsan@example.com", "createdAt": 1700000000001 }
  ],
  "example.com": [
    { "id": "test_d1", "label": "站点-账号", "value": "test_user_001", "createdAt": 1700000000002 },
    { "id": "test_d2", "label": "站点-多行备注", "value": "第一行\n第二行\n第三行", "createdAt": 1700000000003 }
  ]
}
```

> 导入方式：打开设置页 → 导入/导出 → 粘贴上方 JSON → 点击"导入"

### 测试后：重置数据
- **清空所有数据**：设置页 → 导入区粘贴 `{}` → 点击"导入" → 确认覆盖
- **或**：打开 Tampermonkey 仪表盘 → 存储 → 删除 `autofill_helper_data` 键

## 决策记录

> 记录本测试方案形成过程中的关键决策，便于日后调整时追溯原因。

| 决策点 | 结论 | 备注 |
|--------|------|------|
| 测试方式 | 手工测试 | 脚本深度依赖浏览器 DOM / Shadow DOM / TM API，自动化搭建成本高 |
| 文件格式 | Markdown 表格，存入项目仓库根目录 | 与代码一起版本化，发版前 checklist 式过一遍，轻量可追溯 |
| 文件命名 | 贴合产品名 AutoFill Helper，使用描述性后缀 | 避免 _B/_C 等无语义后缀，文件名自解释 |
| 版本数量 | 两份：STANDARD（标准版）+ CHECKLIST（实用版） | STANDARD 作规格参考文档；CHECKLIST 作日常执行文档，职责分离 |
| 表格列设计 | STANDARD：前置条件 + 操作步骤 + 预期结果；CHECKLIST：预期结果 + 优先级 + 结果 + 备注 | 步骤列写起来耗时，仅保留在 STANDARD；CHECKLIST 保持轻量 |
| 优先级 | 仅 CHECKLIST 版有，P0/P1/P2 三级，P0 共 15 条 | STANDARD 定位为规格文档，不做执行分级；优先级只维护一处避免不一致 |
